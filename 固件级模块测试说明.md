# 固件级模块测试说明

本文档说明当前工程内置的固件级模块测试方法。测试入口位于 `Core/Src/main.c`，测试实现位于 `Core/Src/medicine_car_tests.c`，测试开关位于 `Core/Inc/medicine_car_config.h`。

## 1. 测试模式总开关

通过修改 `Core/Inc/medicine_car_config.h` 中的 `MED_CAR_TEST_MODE` 选择要烧录的测试模式：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_NONE
```

可选值如下：

| 宏值 | 功能 |
|---|---|
| `MED_CAR_TEST_MODE_NONE` | 正常主程序模式 |
| `MED_CAR_TEST_MODE_MOTOR` | 电机方向和 PWM 测试 |
| `MED_CAR_TEST_MODE_GRAY` | 灰度传感器读数测试 |
| `MED_CAR_TEST_MODE_GRAY_TRACE` | 灰度循迹测试 |
| `MED_CAR_TEST_MODE_LED_BEEP` | LED 和蜂鸣器测试 |
| `MED_CAR_TEST_MODE_ROUTE1` | 路线 1 实车测试 |
| `MED_CAR_TEST_MODE_ROUTE2` | 路线 2 实车测试 |
| `MED_CAR_TEST_MODE_ENCODER` | 编码器计数测试 |
| `MED_CAR_TEST_MODE_WHEEL_MATCH` | 左右轮等速 PWM 校准测试 |
| `MED_CAR_TEST_MODE_VISION_UART` | OpenMV 视觉串口请求-响应测试 |
| `MED_CAR_TEST_MODE_DRUG_SENSOR` | 红外对管药品检测测试 |
| `MED_CAR_TEST_MODE_SENSOR_TURNS` | 传感器引导转弯测试（左转/右转/掉头） |
| `MED_CAR_TEST_MODE_SENSOR_DOOR` | 传感器虚线门口检测测试 |
| `MED_CAR_TEST_MODE_CROSS_TURN` | 巡线到十字后转弯调参测试 |
| `MED_CAR_TEST_MODE_ROUTE3_8` | 路线 3-8 动态目标完整测试 |

当 `MED_CAR_TEST_MODE != MED_CAR_TEST_MODE_NONE` 时，程序会在初始化完成后进入 `MedicineCar_RunFirmwareTestLoop()`，不会进入正常的 `MedicineCar_Step()` 主业务循环。

## 2. 串口观察方式

测试信息通过 `u2_printf()` 输出到 `USART1`：

| 项目 | 配置 |
|---|---|
| 串口 | USART1 |
| 波特率 | 115200 |
| TX | PB14 |
| RX | PB15 |
| 数据格式 | 8N1 |

接线方式：

```text
STM32 PB14 / USART1_TX -> USB-TTL RX
STM32 PB15 / USART1_RX -> USB-TTL TX
STM32 GND              -> USB-TTL GND
```

电脑串口助手选择对应 COM 口，波特率设置为 `115200`。

## 3. 灰度循迹测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_GRAY_TRACE
```

相关参数：

```c
#define MED_CAR_TEST_GRAY_TRACE_PWM      1800
#define MED_CAR_TEST_GRAY_TRACE_DISTANCE 3000U
#define MED_CAR_TEST_GRAY_TRACE_PAUSE_MS 1000U
```

该测试会真正调用 `xunxian()`，让小车按当前灰度循迹算法低速前进一小段。每轮测试流程为：

1. 打印循迹前 8 路灰度状态。
2. 执行 `xunxian(MED_CAR_TEST_GRAY_TRACE_DISTANCE, MED_CAR_TEST_GRAY_TRACE_PWM)`。
3. 停车。
4. 打印循迹后 8 路灰度状态。
5. 暂停后继续下一轮。

串口示例：

```text
GRAY BEFORE TRACE: 0 0 1 0 0 0 0 0
GRAY AFTER TRACE: 0 0 0 0 1 0 0 0
Gray trace segment done
```

测试前请先完成电机方向测试和灰度传感器读数测试。首次测试建议把 `MED_CAR_TEST_GRAY_TRACE_PWM` 设得偏低，并让小车在短直线黑线上验证是否能自动修正方向。

注意：当前 `xunxian()` 的距离退出条件依赖编码器计数。工程默认 `MED_CAR_ENABLE_ENCODER` 为 `1U`，使用 TIM1/TIM8 真实计数；若临时改为 `0U`，程序会回退到虚拟计数，此时只能验证循迹动作和方向修正，不能证明真实编码器距离闭环已经正常。

## 4. LED 和蜂鸣器测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_LED_BEEP
```

相关参数：

```c
#define MED_CAR_TEST_LED_BEEP_STEP_MS   500U
```

测试流程会循环执行：

1. LED 亮、蜂鸣器关。
2. LED 灭、蜂鸣器响。
3. LED 亮、蜂鸣器响。
4. LED 灭、蜂鸣器关。

当前代码中 `MedicineCar_SetRedLed()` 和 `MedicineCar_SetGreenLed()` 都使用 `CAR_LED`，实际硬件上暂时无法区分红灯和绿灯；`MedicineCar_SetYellowLed()` 使用 `CAR_BEEP`，当前等同于蜂鸣器控制。

## 5. 路线 1 测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_ROUTE1
```

该测试涵盖从视觉识别到返回起点的**完整流程**：

1. 上电初始化后，等待 OpenMV 视觉模块通过 USART3 识别并发送目标数字。
2. 串口打印 `Waiting for vision recognition...`，轮询等待 `N` 变为非零值。
3. 视觉模块识别到数字（如 `1`）后，`MedicineCar_Step()` 自动设置 `aim = N` 并触发路线执行。
4. 当 `MED_CAR_USE_SENSOR_GUIDED_TURNS=1` 时（默认开启），使用传感器引导流程：
   - `xunxian()` → 按编码器距离和灰度循迹到目标路口附近
   - `sensor_turn_left()` → 先按编码器转过最小角度，再用灰度传感器对准目标支路线
   - `xunxian_until_door()` → 巡线前进，检测虚线门口区域停车
   - `wait_delivery_done()` → 停车亮红灯，等待人工卸药
   - `sensor_diaotou()` → 传感器引导掉头 180°
   - `xunxian()` → 按编码器距离和灰度循迹返回主路
   - `sensor_turn_right()` → 先按编码器转过最小角度，再用灰度传感器右转回主路
   - `finish_at_home()` → 返回起点，点亮绿灯
5. 完成后停车并停留在空闲循环，不会反复执行。

测试前应确认：
1. 电机方向和 PWM 测试已通过。
2. 灰度传感器读数测试已通过。
3. OpenMV 视觉模块已连接 USART3，能正常识别数字 1。
4. 药品已装载（红外对管检测到遮挡）。
5. 小车放在路线 1 起点，车头朝向病房方向。
6. 电池电量足够，场地周围留有安全距离。

## 6. 路线 2 测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_ROUTE2
```

测试流程与路线 1 相同，区别在于：
- 视觉模块需识别数字 `2`。
- 转弯方向对称：第一个路口**右转**入室，返回时**左转**回主路。

完整流程：等待视觉识别 → 路口检测停车 → 传感器右转 → 虚线检测停车 → 卸药 → 传感器掉头 → 传感器检测路口 → 传感器左转 → 返回起点亮绿灯。

测试前提与路线 1 相同，小车放在路线 2 起点。

## 7. 传感器引导转弯测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_SENSOR_TURNS
```

相关参数：

```c
#define MED_CAR_TEST_SENSOR_TURN_PAUSE_MS   5000U
```

当前 `sensor_turn_left()` / `sensor_turn_right()` 采用“两段式”转弯：

1. `move_forward_straight(MED_CAR_CROSS_ADVANCE_TICKS, MED_CAR_CROSS_ADVANCE_PWM)` 先向前离开十字路口中心。
2. 原地旋转，并累计编码器绝对 tick：`abs(Encoder_Left) + abs(Encoder_Right)`。
3. 未达到最小转角 tick 前，不接受灰度对线结果，避免车身倾斜时被灰度传感器提前“摆正”而不是转弯。
4. 达到最小 tick 后，再按左/右/中心灰度判定找线；连续确认 `MED_CAR_SENSOR_TURN_CONFIRM_CNT` 次后停车。

相关参数：

```c
#define MED_CAR_CROSS_ADVANCE_TICKS           110U
#define MED_CAR_CROSS_ADVANCE_PWM             600
#define MED_CAR_SENSOR_TURN_LEFT_MIN_TICKS    180U
#define MED_CAR_SENSOR_TURN_RIGHT_MIN_TICKS   180U
#define MED_CAR_SENSOR_DIAOTOU_MIN_TICKS      320U
#define MED_CAR_SENSOR_TURN_CONFIRM_CNT       2U
#define MED_CAR_TURN_TIMEOUT_MS               2500U
```

该测试循环执行左转、右转、掉头，每轮间隔 `MED_CAR_TEST_SENSOR_TURN_PAUSE_MS` 暂停。每次转弯后通过 UART 打印结果（OK 或 TIMEOUT）。

测试前将小车放在十字路口中心位置。测试流程：

1. 打印 "Turn LEFT"，执行 `sensor_turn_left()`，打印结果。
2. 暂停 `MED_CAR_TEST_SENSOR_TURN_PAUSE_MS`。
3. 打印 "Turn RIGHT"，执行 `sensor_turn_right()`，打印结果。
4. 暂停 `MED_CAR_TEST_SENSOR_TURN_PAUSE_MS`。
5. 打印 "DIAOTOU"，执行 `sensor_diaotou()`，打印结果。
6. 暂停 `MED_CAR_TEST_SENSOR_TURN_PAUSE_MS`，循环。

调参建议：

| 现象 | 优先调整 |
|---|---|
| 车身只是被摆正，没有真正转入支路 | 增大 `MED_CAR_SENSOR_TURN_LEFT_MIN_TICKS` 或 `MED_CAR_SENSOR_TURN_RIGHT_MIN_TICKS` |
| 转过头才停 | 减小对应 `*_MIN_TICKS`，或降低转弯 PWM |
| 经常 TIMEOUT | 增大 `MED_CAR_TURN_TIMEOUT_MS`，或检查灰度判定通道是否正确 |
| 转弯前前冲太多 | 减小 `MED_CAR_CROSS_ADVANCE_TICKS` 或 `MED_CAR_CROSS_ADVANCE_PWM` |

此测试用于独立校准编码器最小转角、灰度对线判定和旋转 PWM。

## 8. 巡线到十字后转弯调参测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_CROSS_TURN
```

相关参数：

```c
#define MED_CAR_TEST_CROSS_TURN_PWM              2500
#define MED_CAR_TEST_CROSS_TURN_CYCLES              3U
#define MED_CAR_TEST_CROSS_TURN_MAX_TICKS       30000U
#define MED_CAR_TEST_CROSS_TURN_PAUSE_MS         2000U
```

该测试用于实车调 `xunxian()` 到十字路口后的左转/右转动作参数，不输出 `distance_ticks` 建议值。`MED_CAR_TEST_CROSS_TURN_MAX_TICKS` 只作为巡线保护上限，防止没有识别到十字时一直前进。

测试逻辑：

1. 将小车放在十字路口前方的巡线路径上，并让车头对准黑线。
2. 每次动作开始前串口提示 `start in 3s`，3 秒后执行 `xunxian(MED_CAR_TEST_CROSS_TURN_MAX_TICKS, MED_CAR_TEST_CROSS_TURN_PWM)`。
3. 检测到十字后停车，并打印本段 `L/R/sum/total_abs/gap`，这些值只用于观察，不作为自动标定结果。
4. LEFT 动作命中十字后调用 `sensor_turn_left()`，RIGHT 动作命中十字后调用 `sensor_turn_right()`。
5. 如果巡线达到 `MED_CAR_TEST_CROSS_TURN_MAX_TICKS` 仍未检测到十字，本次动作标记为 `LIMIT`，跳过转弯。
6. 每个 cycle 依次执行 LEFT 和 RIGHT，各动作之间暂停 `MED_CAR_TEST_CROSS_TURN_PAUSE_MS`。

示例输出：

```text
Cycle 1 LEFT approach CROSS: L=2100 R=2050 sum=4150 total_abs=4150 gap=50
Cycle 1 turn LEFT result: OK
Cycle 1 RIGHT approach CROSS: L=2080 R=2070 sum=4150 total_abs=4150 gap=10
Cycle 1 turn RIGHT result: OK
```

手工调参时重点观察：

| 现象 | 优先调整 |
|---|---|
| 巡线到不了十字或误判十字 | 灰度阈值、十字判定通道、`MED_CAR_TEST_CROSS_TURN_PWM` |
| 转弯前前冲太多或不够 | `MED_CAR_CROSS_ADVANCE_TICKS` / `MED_CAR_CROSS_ADVANCE_PWM` |
| 车身只是摆正，没有真正转入支路 | `MED_CAR_SENSOR_TURN_LEFT_MIN_TICKS` / `MED_CAR_SENSOR_TURN_RIGHT_MIN_TICKS` |
| 转过头或转不到位 | 对应最小转角 tick、转弯 PWM、`MED_CAR_SENSOR_TURN_CONFIRM_CNT` |

测试前应确认灰度循迹、十字路口检测、编码器计数测试和左右轮等速 PWM 校准测试已经通过。

## 9. OpenMV 视觉串口测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_VISION_UART
```

接线：

```text
STM32 PD8 / USART3_TX -> OpenMV P5 / UART3_RX
STM32 PD9 / USART3_RX -> OpenMV P4 / UART3_TX
STM32 GND             -> OpenMV GND
```

测试流程会每秒通过 USART3 发送一次 `REQ\r\n`。OpenMV 返回 `OK,count,digits` 时，USART1 调试口打印识别到的数字；返回 `ERR,NONE`、格式错误或超时时，打印错误信息。

## 10. 路线 3-8 动态目标测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_ROUTE3_8
```

该测试用于联调药品检测模块、OpenMV 视觉识别模块和路线 3-8 完整配送闭环。上电后小车先停车，通过 USART3 循环发送 `REQ\r\n` 请求视觉结果；只缓存首次出现的 `3..8` 目标数字，忽略 `0/1/2` 和无效响应。目标缓存后继续等待药品检测为 `PRESENT`，再开始路线执行。

测试流程：

1. 停车等待视觉目标，串口打印每次 `VISION OK count=... digits=...` 或错误状态。
2. 缓存 `3..8` 中的第一个目标数字，打印 `CACHED target=...`。
3. 等待红外对管检测到药品，未就位时电机保持停止。
4. 启动 `MedicineCar_RunRoute3To8Test(target)`，在 RP1 到 RP5 每个识别点打印视觉状态、`NumBuff/XBuff`、目标值和匹配方向。
5. 若某个识别点包含目标数字，则按 `XBuff==1` 左转、`XBuff==2` 右转进入病房，完成门口停车、卸药等待、掉头和返回。
6. 若所有识别点都未命中目标，则停车、亮红灯并打印 `ROUTE3_8 FAIL no target matched`。

典型串口输出：

```text
=== ROUTE3_8 DYNAMIC TEST ===
Waiting for vision target 3..8
VISION OK count=2 digits=1,3
CACHED target=3
Target cached=3, waiting for drug PRESENT
DRUG raw=LOW interpreted=PRESENT
Drug PRESENT, start route3_8 target=3
=== ROUTE3_8 TEST START target=3 ===
[RP1] xunxian dist=19500 pwm=5000
[RP1] vision ok=1 status=ok
[RP1] Num=[3,0] X=[2,1] target=3
[RP1] match idx=0 room=3 dir=RIGHT
```

重点调试参数：

| 参数 | 用途 |
|---|---|
| `MED_CAR_VISION_UART_TIMEOUT_MS` / `MED_CAR_VISION_RETRY_COUNT` | OpenMV 请求超时和重试次数 |
| `MED_CAR_TEST_R3_8_VISION_INTERVAL_MS` | 起点等待目标时的视觉轮询间隔 |
| `MED_CAR_TEST_R3_8_DRUG_POLL_MS` | 等药阶段轮询间隔 |
| `MED_CAR_DISTANCE_SECOND_CHECK` / `MED_CAR_DISTANCE_THIRD_CHECK` | 主路识别点距离 |
| `MED_CAR_DISTANCE_R3_8_APPROACH` / `MED_CAR_DISTANCE_R3_8_SHORT` / `MED_CAR_DISTANCE_R3_8_ZHAO_BAI` | 路线 3-8 中段补偿距离 |
| `MED_CAR_R3_8_PWM_MAIN` / `MED_CAR_R3_8_PWM_FAHUI` | 路线 3-8 主路速度 |
| `MED_CAR_CROSS_ADVANCE_TICKS` / `MED_CAR_CROSS_ADVANCE_PWM` | 转弯前离开十字路口中心的前进补偿 |
| `MED_CAR_DOOR_BLACK_MIN` / `MED_CAR_DOOR_CONFIRM_CNT` | 门口虚线检测 |
| `MED_CAR_SENSOR_TURN_*_MIN_TICKS` / `MED_CAR_TURN_*` / `MED_CAR_DIAOTOU_*` | 左转、右转、掉头闭环参数 |
| `MED_CAR_LEFT_PWM_TRIM_NUM` / `MED_CAR_RIGHT_PWM_TRIM_NUM` | 左右轮直线速度校准 |

测试前应先确认：

1. `MED_CAR_TEST_MODE_VISION_UART` 能稳定打印 OpenMV 结果。
2. `MED_CAR_TEST_MODE_DRUG_SENSOR` 已通过，能正确区分 `PRESENT/ABSENT`。
3. `MED_CAR_TEST_MODE_SENSOR_TURNS` 已完成验证；虚线门口检测已通过 `MED_CAR_TEST_MODE_SENSOR_DOOR` 完成验证；十字路口检测已嵌入 `xunxian()` 巡线过程。
4. 小车放在药房起点，车头朝向主路，场地周围留有安全距离。

