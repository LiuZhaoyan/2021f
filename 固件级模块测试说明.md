# 固件级模块测试说明

本文档说明当前工程内置的固件级模块测试方法。测试入口位于 `Core/Src/main.c`，测试实现位于 `Core/Src/medicine_car_tests.c`，测试开关位于 `Core/Inc/medicine_car_config.h`。

## 1. 测试模式总开关

通过修改 `Core/Inc/medicine_car_config.h` 中的 `MED_CAR_TEST_MODE` 选择要烧录的测试模式：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_NONE
```

可选值如下：

| 宏值 | 功能 |
|---|---|
| `MED_CAR_TEST_MODE_NONE` | 正常主程序模式 |
| `MED_CAR_TEST_MODE_MOTOR` | 电机方向和 PWM 测试 |
| `MED_CAR_TEST_MODE_GRAY` | 灰度传感器读数测试 |
| `MED_CAR_TEST_MODE_GRAY_TRACE` | 灰度循迹测试 |
| `MED_CAR_TEST_MODE_LED_BEEP` | LED 和蜂鸣器测试 |
| `MED_CAR_TEST_MODE_ROUTE1` | 路线 1 实车测试 |
| `MED_CAR_TEST_MODE_ROUTE2` | 路线 2 实车测试 |
| `MED_CAR_TEST_MODE_ENCODER` | 编码器计数测试 |

当 `MED_CAR_TEST_MODE != MED_CAR_TEST_MODE_NONE` 时，程序会在初始化完成后进入 `MedicineCar_RunFirmwareTestLoop()`，不会进入正常的 `MedicineCar_Step()` 主业务循环。

## 2. 串口观察方式

测试信息通过 `u2_printf()` 输出到 `USART1`：

| 项目 | 配置 |
|---|---|
| 串口 | USART1 |
| 波特率 | 115200 |
| TX | PB14 |
| RX | PB15 |
| 数据格式 | 8N1 |

接线方式：

```text
STM32 PB14 / USART1_TX -> USB-TTL RX
STM32 PB15 / USART1_RX -> USB-TTL TX
STM32 GND              -> USB-TTL GND
```

电脑串口助手选择对应 COM 口，波特率设置为 `115200`。

## 3. 电机方向和 PWM 测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_MOTOR
```

相关参数：

```c
#define MED_CAR_TEST_MOTOR_PWM          2000
#define MED_CAR_TEST_MOTOR_RUN_MS       1200U
#define MED_CAR_TEST_MOTOR_STOP_MS      800U
```

测试流程会循环执行：

1. 左轮正转。
2. 左轮反转。
3. 右轮正转。
4. 右轮反转。
5. 双轮前进。
6. 双轮后退。

上车前请先把车轮架空，确认每个阶段轮子方向是否符合串口打印。若某个电机方向相反，可修改：

```c
#define MED_CAR_LEFT_MOTOR_INVERT       1U
#define MED_CAR_RIGHT_MOTOR_INVERT      1U
```

只反哪个电机，就只改对应宏。

## 4. 灰度传感器测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_GRAY
```

相关参数：

```c
#define MED_CAR_TEST_GRAY_PRINT_MS      500U
```

串口会周期打印：

```text
GRAY: 0 1 0 0 1 0 0 0
```

8 个数字依次对应 `IO1` 到 `IO8`。当前配置中：

```c
#define MED_CAR_GRAY_BLACK_LEVEL        1U
```

因此打印值为 `1` 表示检测到黑线，`0` 表示未检测到黑线。测试时可以让灰度模块依次经过黑线和白底，观察对应通道是否变化。

## 5. 灰度循迹测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_GRAY_TRACE
```

相关参数：

```c
#define MED_CAR_TEST_GRAY_TRACE_PWM      1800
#define MED_CAR_TEST_GRAY_TRACE_DISTANCE 3000U
#define MED_CAR_TEST_GRAY_TRACE_PAUSE_MS 1000U
```

该测试会真正调用 `xunxian()`，让小车按当前灰度循迹算法低速前进一小段。每轮测试流程为：

1. 打印循迹前 8 路灰度状态。
2. 执行 `xunxian(MED_CAR_TEST_GRAY_TRACE_DISTANCE, MED_CAR_TEST_GRAY_TRACE_PWM)`。
3. 停车。
4. 打印循迹后 8 路灰度状态。
5. 暂停后继续下一轮。

串口示例：

```text
GRAY BEFORE TRACE: 0 0 1 0 0 0 0 0
GRAY AFTER TRACE: 0 0 0 0 1 0 0 0
Gray trace segment done
```

测试前请先完成电机方向测试和灰度传感器读数测试。首次测试建议把 `MED_CAR_TEST_GRAY_TRACE_PWM` 设得偏低，并让小车在短直线黑线上验证是否能自动修正方向。

注意：当前 `xunxian()` 的距离退出条件依赖编码器计数。若 `MED_CAR_ENABLE_ENCODER` 为 `0U`，程序会使用虚拟计数，因此该测试可以用于验证循迹动作和方向修正，但不能证明真实编码器距离闭环已经正常。

## 6. LED 和蜂鸣器测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_LED_BEEP
```

相关参数：

```c
#define MED_CAR_TEST_LED_BEEP_STEP_MS   500U
```

测试流程会循环执行：

1. LED 亮、蜂鸣器关。
2. LED 灭、蜂鸣器响。
3. LED 亮、蜂鸣器响。
4. LED 灭、蜂鸣器关。

当前代码中 `MedicineCar_SetRedLed()` 和 `MedicineCar_SetGreenLed()` 都使用 `CAR_LED`，实际硬件上暂时无法区分红灯和绿灯；`MedicineCar_SetYellowLed()` 使用 `CAR_BEEP`，当前等同于蜂鸣器控制。

## 7. 路线 1 测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_ROUTE1
```

初始化完成后程序会自动执行一次路线 1，相当于：

```c
MedicineCar_RequestRun(1);
MedicineCar_Step();
```

执行完成后会停车并停留在空闲循环，不会反复执行路线 1。测试前应确认：

1. 电机方向已经通过。
2. 灰度读数已经通过。
3. 小车放在路线 1 起点。
4. 电池电量足够，场地周围留有安全距离。

## 8. 路线 2 测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_ROUTE2
```

初始化完成后程序会自动执行一次路线 2，相当于：

```c
MedicineCar_RequestRun(2);
MedicineCar_Step();
```

执行完成后会停车并停留在空闲循环，不会反复执行路线 2。测试前提与路线 1 相同。

## 9. 编码器计数测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_ENCODER
```

相关参数：

```c
#define MED_CAR_TEST_ENCODER_PWM        2000
#define MED_CAR_TEST_ENCODER_RUN_MS     2000U
#define MED_CAR_TEST_ENCODER_SAMPLE_MS  200U
```

测试流程会循环执行：

1. 清零 `Encoder_Left` 和 `Encoder_Right`。
2. 双轮以固定 PWM 前进。
3. 每隔 `MED_CAR_TEST_ENCODER_SAMPLE_MS` 打印一次左右编码器计数。
4. 到达 `MED_CAR_TEST_ENCODER_RUN_MS` 后停车。

串口示例：

```text
Encoder 200 ms: L=80 R=80 SpeedL=80 SpeedR=80
Encoder 400 ms: L=160 R=160 SpeedL=160 SpeedR=160
```

注意：当前 `MED_CAR_ENABLE_ENCODER` 默认是 `0U`，此时程序使用虚拟计数，打印增长只能说明测试循环在运行，不能证明真实编码器硬件已接入。要测试真实编码器，需要完成编码器定时器/引脚配置、实现 `Read_Encoder()`，再将：

```c
#define MED_CAR_ENABLE_ENCODER          1U
```

若启用真实编码器后计数始终为 0，应优先检查编码器接线、定时器编码器模式、A/B 相方向和 `Read_Encoder()` 的定时器编号映射。

## 10. 恢复正常程序

完成测试后改回：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_NONE
```

重新编译并烧录，程序会恢复正常主循环：

```c
MedicineCar_Step();
```
