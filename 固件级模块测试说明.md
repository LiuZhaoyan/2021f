# 固件级模块测试说明

本文档说明当前工程内置的固件级模块测试方法。测试入口位于 `Core/Src/main.c`，测试实现位于 `Core/Src/medicine_car_tests.c`，测试开关位于 `Core/Inc/medicine_car_config.h`。

## 1. 测试模式总开关

通过修改 `Core/Inc/medicine_car_config.h` 中的 `MED_CAR_TEST_MODE` 选择要烧录的测试模式：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_NONE
```

可选值如下：

| 宏值 | 功能 |
|---|---|
| `MED_CAR_TEST_MODE_NONE` | 正常主程序模式 |
| `MED_CAR_TEST_MODE_MOTOR` | 电机方向和 PWM 测试 |
| `MED_CAR_TEST_MODE_GRAY` | 灰度传感器读数测试 |
| `MED_CAR_TEST_MODE_GRAY_TRACE` | 灰度循迹测试 |
| `MED_CAR_TEST_MODE_LED_BEEP` | LED 和蜂鸣器测试 |
| `MED_CAR_TEST_MODE_ROUTE1` | 路线 1 实车测试 |
| `MED_CAR_TEST_MODE_ROUTE2` | 路线 2 实车测试 |
| `MED_CAR_TEST_MODE_ENCODER` | 编码器计数测试 |
| `MED_CAR_TEST_MODE_WHEEL_MATCH` | 左右轮等速 PWM 校准测试 |
| `MED_CAR_TEST_MODE_DRUG_SENSOR` | 红外对管药品检测测试 |
| `MED_CAR_TEST_MODE_SENSOR_DOOR` | 传感器虚线门口检测测试 |
| `MED_CAR_TEST_MODE_ROUTE3_8` | 路线 3-8 动态目标完整测试 |
| `MED_CAR_TEST_MODE_WIGGLE_CALIB` | RP2 四数字 wiggle 扫描标定测试 |
| `MED_CAR_TEST_MODE_RP2` | 从 RP2 前巡线到匹配并完成左右转的专项流程测试 |

当 `MED_CAR_TEST_MODE != MED_CAR_TEST_MODE_NONE` 时，程序会在初始化完成后进入 `MedicineCar_RunFirmwareTestLoop()`，不会进入正常的 `MedicineCar_Step()` 主业务循环。

## 2. 串口观察方式

测试信息通过 `u2_printf()` 输出到 `USART1`：

| 项目 | 配置 |
|---|---|
| 串口 | USART1 |
| 波特率 | 115200 |
| TX | PB14 |
| RX | PB15 |
| 数据格式 | 8N1 |

接线方式：

```text
STM32 PB14 / USART1_TX -> USB-TTL RX
STM32 PB15 / USART1_RX -> USB-TTL TX
STM32 GND              -> USB-TTL GND
```

电脑串口助手选择对应 COM 口，波特率设置为 `115200`。

## 3. 灰度循迹测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_GRAY_TRACE
```

相关参数：

```c
#define MED_CAR_TEST_GRAY_TRACE_PWM      1800
#define MED_CAR_TEST_GRAY_TRACE_DISTANCE 3000U
#define MED_CAR_TEST_GRAY_TRACE_PAUSE_MS 1000U
```

该测试会真正调用 `xunxian()`，让小车按当前灰度循迹算法低速前进一小段。每轮测试流程为：

1. 打印循迹前 8 路灰度状态。
2. 执行 `xunxian(MED_CAR_TEST_GRAY_TRACE_DISTANCE, MED_CAR_TEST_GRAY_TRACE_PWM)`。
3. 停车。
4. 打印循迹后 8 路灰度状态。
5. 暂停后继续下一轮。

串口示例：

```text
GRAY BEFORE TRACE: 0 0 1 0 0 0 0 0
GRAY AFTER TRACE: 0 0 0 0 1 0 0 0
Gray trace segment done
```

测试前请先完成电机方向测试和灰度传感器读数测试。首次测试建议把 `MED_CAR_TEST_GRAY_TRACE_PWM` 设得偏低，并让小车在短直线黑线上验证是否能自动修正方向。

注意：当前 `xunxian()` 的距离退出条件依赖编码器计数。工程默认 `MED_CAR_ENABLE_ENCODER` 为 `1U`，使用 TIM1/TIM8 真实计数；若临时改为 `0U`，程序会回退到虚拟计数，此时只能验证循迹动作和方向修正，不能证明真实编码器距离闭环已经正常。

## 4. LED 和蜂鸣器测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_LED_BEEP
```

相关参数：

```c
#define MED_CAR_TEST_LED_BEEP_STEP_MS   500U
```

测试流程会循环执行：

1. LED 亮、蜂鸣器关。
2. LED 灭、蜂鸣器响。
3. LED 亮、蜂鸣器响。
4. LED 灭、蜂鸣器关。

当前代码中 `MedicineCar_SetRedLed()` 和 `MedicineCar_SetGreenLed()` 都使用 `CAR_LED`，实际硬件上暂时无法区分红灯和绿灯；`MedicineCar_SetYellowLed()` 使用 `CAR_BEEP`，当前等同于蜂鸣器控制。

## 5. 路线 1 测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_ROUTE1
```

该测试涵盖从视觉识别到返回起点的**完整流程**：

1. 测试入口打开全新的视觉读取窗口，等待 OpenMV 主动发送首个包含 `1..2` 的完整有效帧。
2. 目标缓存后关闭视觉窗口，再等待药品传感器变为 `PRESENT`。
3. 调用 `MedicineCar_RunRoute12Test(target)` 执行路线表；路线本身不再经过正常 `MedicineCar_Step()` 自动触发。
4. 到达固定岔路后按目标执行传感器引导转弯，随后检测门口、等待卸药并按返回栈返航。
5. 完成后停车并停留在测试空闲循环，不会反复执行。

测试前应确认：
1. 电机方向和 PWM 测试已通过。
2. 灰度传感器读数测试已通过。
3. OpenMV 视觉模块已连接 USART3，能正常识别数字 1。
4. 药品已装载（红外对管检测到遮挡）。
5. 小车放在路线 1 起点，车头朝向病房方向。
6. 电池电量足够，场地周围留有安全距离。

## 6. 路线 2 测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_ROUTE2
```

测试流程与路线 1 相同，区别在于：
- 视觉模块需识别数字 `2`。
- 转弯方向对称：第一个路口**右转**入室，返回时**左转**回主路。

完整流程：等待视觉识别 → 路口检测停车 → 传感器右转 → 虚线检测停车 → 卸药 → 传感器掉头 → 传感器检测路口 → 传感器左转 → 返回起点亮绿灯。

测试前提与路线 1 相同，小车放在路线 2 起点。

## 7. 路线 3-8 动态目标测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_ROUTE3_8
```

该测试用于联调药品检测模块、OpenMV 视觉识别模块和路线 3-8 完整配送闭环。OpenMV 约每 50 ms 主动发送 `AA BB left right CC`；测试入口按读取窗口缓存首个出现的 `3..8` 目标数字，忽略不含目标范围的有效帧和所有不完整/非法帧。目标缓存后关闭窗口并等待药品检测为 `PRESENT`，再开始路线执行。

测试流程：

1. 停车打开目标读取窗口，串口按 `[函数][阶段]` 打印完整帧或等待超时。
2. 按画面从左到右缓存 `3..8` 中的第一个目标数字，打印 `TARGET_CACHED`。
3. 等待红外对管检测到药品，未就位时电机保持停止。
4. 启动 `MedicineCar_RunRoute3To8Test(target)`；每个路段开始先清空旧视觉缓存，并打印路段、帧、`NumBuff/XBuff`、目标值和匹配方向。
5. 若某个识别点包含目标数字，则按 `XBuff==1` 左转、`XBuff==2` 右转进入病房，完成门口停车、卸药等待、掉头和返回。
6. 某识别点无帧或未命中时直行驶出并继续下一识别点；全部识别点未命中时停车、亮红灯并打印路线失败。

典型串口输出：

```text
[debug_route3_8_test_loop][START] ROUTE3_8 DYNAMIC TEST
[wait_for_test_target][WAIT_TARGET] waiting indefinitely for target 3..8
[print_vision_frame][VISION_FRAME] left=1 right=3 timestamp=...
[wait_for_test_target][TARGET_CACHED] target=3
[debug_route3_8_test_loop][ROUTE_START] drug=PRESENT target=3
[route_run][SEG_BEGIN] index=1 type=FORK distance=4000 cache=cleared
[route_run][FORK_FRAME] left=3 right=0 timestamp=...
[check_match_and_turn][MATCH] aim=3 Num=[3,0] X=[1,0] side=1
```

重点调试参数：

| 参数 | 用途 |
|---|---|
| `MED_CAR_VISION_UART_TIMEOUT_MS` | 单个视觉读取窗口的等待超时；启动目标阶段超时后会重新开窗继续等待 |
| `MED_CAR_TEST_R3_8_VISION_INTERVAL_MS` | 起点等待目标时的视觉轮询间隔 |
| `MED_CAR_TEST_R3_8_DRUG_POLL_MS` | 等药阶段轮询间隔 |
| `MED_CAR_DISTANCE_SECOND_CHECK` / `MED_CAR_DISTANCE_R3_8_SHORT` | 路线 3-8 十字路口和相邻识别路段的最大距离 |
| `MED_CAR_TRACE_PWM` | 路线表巡线速度 |
| `MED_CAR_DOOR_BLACK_MIN` / `MED_CAR_DOOR_CONFIRM_CNT` | 门口虚线检测 |
| `MED_CAR_SENSOR_TURN_*_MIN_TICKS` / `MED_CAR_TURN_*` / `MED_CAR_DIAOTOU_*` | 左转、右转、掉头闭环参数 |
| `MED_CAR_LEFT_PWM_TRIM_NUM` / `MED_CAR_RIGHT_PWM_TRIM_NUM` | 左右轮直线速度校准 |

测试前应先确认：

1. OpenMV 视觉串口链路已完成验证。
2. `MED_CAR_TEST_MODE_DRUG_SENSOR` 已通过，能正确区分 `PRESENT/ABSENT`。
3. 巡线到十字后的转弯动作已完成调参；虚线门口检测已通过 `MED_CAR_TEST_MODE_SENSOR_DOOR` 完成验证；十字路口检测已嵌入 `xunxian()` 巡线过程。
4. 小车放在药房起点，车头朝向主路，场地周围留有安全距离。

## 8. RP2 四数字 wiggle 扫描标定测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_WIGGLE_CALIB
```

该测试用于标定路线 3-8 中 RP2 识别点的车头微摆角度。RP2 处有 4 个房间数字，当前扫描逻辑通过左右两个外侧视角收集数字。标定测试循环执行“左摆 → 右摆回中”动作，在串口打印每次摆动的编码器增量，供手工调整 `MED_CAR_RP2_WIGGLE_TICKS`。

相关参数：

```c
#define MED_CAR_RP2_WIGGLE_TICKS             300U   /* 当前目标编码器总增量 */
#define MED_CAR_RP2_WIGGLE_LEFT_PWM_LEFT    -1000   /* 左摆时左轮 PWM */
#define MED_CAR_RP2_WIGGLE_LEFT_PWM_RIGHT    1000   /* 左摆时右轮 PWM */
#define MED_CAR_RP2_WIGGLE_RIGHT_PWM_LEFT    1000   /* 右摆时左轮 PWM */
#define MED_CAR_RP2_WIGGLE_RIGHT_PWM_RIGHT  -1000   /* 右摆时右轮 PWM */
#define MED_CAR_RP2_WIGGLE_TIMEOUT_MS        5000U  /* 摆动超时保护 */
#define MED_CAR_TEST_WIGGLE_PAUSE_MS         2000U  /* 每轮间隔 */
```

测试流程：

1. 将小车放在平地上（无需黑线）。
2. 串口打印当前 PWM 和目标 ticks 配置。
3. 3 秒后进入循环：
   - 左摆 `MED_CAR_RP2_WIGGLE_TICKS` ticks，打印编码器前后值和 delta 绝对值之和。
   - 右摆 `MED_CAR_RP2_WIGGLE_TICKS` ticks 回中，打印编码器 delta。
   - 暂停 2 秒后重复。

示例输出：

```text
=== WIGGLE CALIBRATION TEST ===
PWM: L=(-1000,1000) R=(1000,-1000)
Target ticks: 300
Cycle: L -> print -> R(back) -> print -> pause
L: before enc L=0 R=0
L: after enc L=-80 R=75 | delta L=-80 R=75 sum=155
R: before enc L=-80 R=75
R: after enc L=5 R=-3 | delta L=85 R=-78 sum=163
--- pause 2000ms ---
```

手工调参指导：

| 现象 | 调整 |
|---|---|
| 物理摆角太小（不足 10 度） | 增大 `MED_CAR_RP2_WIGGLE_TICKS` |
| 物理摆角太大（超过 20 度） | 减小 `MED_CAR_RP2_WIGGLE_TICKS` |
| 摆动过快难以控制 | 降低 PWM 绝对值（如改为 ±500） |
| 左右摆角不对称 | 分别调整左右 PWM 值 |

标定完成后将 `MED_CAR_RP2_WIGGLE_TICKS` 的最终值写入 `medicine_car_config.h`，然后切换到 `MED_CAR_TEST_MODE_ROUTE3_8` 进行集成验证。

## 9. RP2 巡线、识别、匹配和转弯专项测试

配置：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_RP2
```

该测试只隔离 RP2 一段流程：从 RP2 前方黑线开始巡线，数字首帧触发停车，执行当前 RP2 左右摆动扫描，将识别结果与目标房间号匹配，并调用正式的传感器引导左转或右转。测试入口调用生产代码中的 `rp2_scan_before_fork()` 和 `check_match_and_turn()`，没有复制另一套测试算法。

### 9.1 参数来源

本测试没有新增测试专用运动或视觉参数，直接使用当前正式流程的宏：

| 流程 | 直接复用的当前参数 |
|---|---|
| RP2 前巡线距离 | `MED_CAR_DISTANCE_R3_8_SHORT` |
| 巡线 PWM | `MED_CAR_TEST_GRAY_TRACE_PWM`，与当前 `route_run()` 路线 3-8 调用一致 |
| 稳定延时 | `MED_CAR_RP2_SCAN_SETTLE_MS` |
| 左右摆动 | `MED_CAR_RP2_WIGGLE_*`、`MED_CAR_RP2_WIGGLE_TICKS`、`MED_CAR_RP2_WIGGLE_TIMEOUT_MS` |
| 外侧视角等帧 | `MED_CAR_RP2_SCAN_TIMEOUT_MS` |
| 扫描后巡线到岔路 | `MED_CAR_RP2_TO_FORK_MAX_DISTANCE` |
| 匹配提示 | `MED_CAR_MATCH_BEEP_MS` |
| 左右转 | `MED_CAR_TURN_*`、`MED_CAR_SENSOR_TURN_*` |

后续调整正式 RP2 参数后，专项测试会自动使用新值，不需要同步维护测试参数。

### 9.2 实车操作

1. 上电前先取下药品，让药品传感器处于 `ABSENT`。
2. 上电后保持小车停车，将目标卡片 `3..8` 中的一个数字放到摄像头前。
3. USART1 打印 `[RP2 TEST] target cached=...` 后，目标值已经锁定，不会再被路口数字覆盖。
4. 将小车放到 RP2 路口前的主路黑线上，车头朝向 RP2，保证前方巡线距离不超过 `MED_CAR_DISTANCE_R3_8_SHORT`。
5. 放入药品，使传感器从 `ABSENT` 变为 `PRESENT`。这个上升过程是专项测试的人工启动信号。
6. 小车开始巡线；测试只执行一次。匹配并完成传感器引导转弯后打印 `PASS`，任一阶段失败则停车并打印 `FAIL`。

如果上电时药品已经存在，程序会停在 `waiting for drug ABSENT`，必须先取下药品，再按上述步骤放回，避免尚未摆好小车就自动启动。

### 9.3 串口阶段和判读

典型成功输出：

```text
[debug_rp2_test_loop][START] RP2 PROCESS TEST
[wait_for_test_target][TARGET_CACHED] target=5
[MedicineCar_RunRp2Test][START] target=5 approach=4000 to_fork=4000 pwm=1500
[rp2_scan_before_fork][APPROACH] stopped reason=VISION enc=(...,...)
[rp2_scan_before_fork][TRIGGER] left=3 right=5 timestamp=...
[rp2_wiggle_with_log][LEFT_SCAN] OK enc=(...,... ) total=...
[rp2_wait_view][LEFT_VISION] frame left=3 right=5 timestamp=...
[rp2_scan_before_fork][TO_FORK] FORK enc=(...,...)
[rp2_scan_before_fork][BUFFER] aim=5 Num=[5,...] X=[1,...]
[check_match_and_turn][MATCH] aim=5 Num=[5,...] X=[1,...] side=1
[check_match_and_turn][TURN] LEFT command
[MedicineCar_RunRp2Test][RESULT] PASS target=5 motion_ok=1 match_result=1
```

关键失败输出：

| 输出 | 说明和排查方向 |
|---|---|
| `stopped reason=FORK` | 灰度先检测到岔路，但数字首帧尚未到达；检查摄像头视野、识别距离和串口帧时序。 |
| `stopped reason=LIMIT` | 到达距离或循环保护上限仍未识别数字，也未满足岔路条件。 |
| `WIGGLE ... TIMEOUT` | 对应摆动在 `MED_CAR_RP2_WIGGLE_TIMEOUT_MS` 内没有达到编码器 ticks；检查摆动 PWM、静摩擦和编码器。 |
| `TIMEOUT: no complete valid frame` | 新读取窗口内没有收到完整有效帧；OpenMV 发送的 `0,0` 不算有效帧。 |
| `TO_FORK ... LIMIT` | 左右摆回正后继续巡线，但在 `MED_CAR_RP2_TO_FORK_MAX_DISTANCE` 内没有检测到岔路。 |
| `MATCH ... side=0` | 扫描收到了数字，但最终集合中没有目标 `aim`。 |
| `TURN ... TIMEOUT` | 已匹配并发出转弯电机命令，但未达到编码器最小角度或灰度连续对线条件。 |

专项测试失败时不会执行集成路线的直行兜底。左右摆扫描完成后，小车先继续巡线到传感器确认的岔路位置；若巡线超限则立即停车并判失败。

## 10. 切回空闲主程序

完成测试后改回：

```c
#define MED_CAR_TEST_MODE               MED_CAR_TEST_MODE_NONE
```

当前 `MedicineCar_Step()` 只保持停车和平台服务，不会自动启动配送路线；待三项路线测试通过后再把路线表接回正常主业务。

重新编译并烧录，程序会恢复正常主循环：

```c
MedicineCar_Step();
```
