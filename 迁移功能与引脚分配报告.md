# STM32H743 药品配送小车迁移功能与引脚分配报告

## 1. 项目状态

- 目标芯片：STM32H743VITx，封装 LQFP100。
- 当前工程骨架：STM32CubeMX/HAL + MDK-ARM。
- 原 `CODE/` 目录已删除，旧 STM32F407ZG 标准库工程不再保留在项目中。
- 当前编译入口位于 `Core/Src/main.c`，主循环调用 `MedicineCar_Step()`。
- Keil 工程 `MDK-ARM/2021f.uvprojx` 已加入 `medicine_car_app.c` 和 `medicine_car_platform.c`，未引用 F4 标准库、`stm32f4xx` 或 `FWLIB`。

## 3. 功能说明

### 3.1 电机控制

- 使用当前 H743 工程已有的 `TIM3_CH1` 和 `TIM3_CH2` 输出双路 PWM。
- `Load(moto1, moto2)` 接收旧 CODE 风格的正负 PWM 值：
  - 正值：正转。
  - 负值：反转。
  - 0：停止。
- PWM 输入范围限制为 `-8000` 到 `8000`，再映射到 H743 当前 `TIM3` 周期 `1999`。

### 3.2 循迹与灰度传感器

- 当前灰度模块按感为官方串行通讯方式接入，信号为 `CLK`、`DAT`。
- `CLK` 由主控输出，`DAT` 由传感器输出；平台层每帧读取 8 bit，并缓存到 `gray_sensor_values[0..7]`。
- 串行第 0 bit 对应第 1 路，第 7 bit 对应第 8 路。
- 旧 CODE 上层仍按 `IO1..IO7` 判断颜色/位置；当前只替换底层读取来源，不改变原先颜色判断逻辑。
- `MED_CAR_GRAY_BLACK_LEVEL` 当前为 `1`，即 `DAT=1` 视为检测到黑线。

### 3.3 药品配送业务

- 支持按目标房间号进入不同路线：
  - `aim == 1`：执行一号房路线。
  - `aim == 2`：执行二号房路线。
  - `aim > 2`：执行扩展路线流程。
- 识别结果通过 `MedicineCar_SetRecognizedNumber()` 或全局变量 `N` 输入。
- 启动路线可通过 `MedicineCar_RequestRun(target_room)` 设置，内部会置位 `Run_Flag`。

### 3.4 编码器计数

- 左轮编码器使用 `TIM1_CH1/CH2`，接线为 `PA8=A 相`、`PA9=B 相`。
- 右轮编码器使用 `TIM8_CH1/CH2`，接线为 `PC6=A 相`、`PC7=B 相`。
- `Read_Speed()` 每次读取硬件计数器增量，并累计到 `Encoder_Left` 和 `Encoder_Right`。
- `Speed_L` 和 `Speed_R` 表示最近一次采样周期的编码器增量；若前进时某侧为负，可修改 `MED_CAR_LEFT_ENCODER_INVERT` 或 `MED_CAR_RIGHT_ENCODER_INVERT`。

### 3.5 未搭建硬件处理

以下硬件相关开关集中在 `Core/Inc/medicine_car_config.h`：

| 开关 | 默认值 | 说明 |
|---|---:|---|
| `MED_CAR_ENABLE_ENCODER` | 1 | 默认使用 TIM1/TIM8 真实编码器计数 |
| `MED_CAR_ENABLE_BLUETOOTH` | 0 | 蓝牙串口暂未接入 |
| `MED_CAR_ENABLE_RECOGNITION_UART` | 0 | 数字识别串口暂未接入 |
| `MED_CAR_ENABLE_TFT` | 0 | TFT 显示暂未接入 |
| `MED_CAR_ENABLE_KEYS` | 0 | 按键暂未接入 |
| `MED_CAR_ENABLE_DRUG_SENSOR` | 0 | 药品检测输入暂未接入，默认认为药品已就绪 |

## 5. 灰度传感器引脚分配

当前 H743 工程已适配感为八路灰度传感器串行通讯：

| 灰度模块信号 | 当前引脚 | CubeMX 标签 | 代码宏 | 方向 |
|---|---|---|---|---|
| CLK | PC0 | `GRAY_CLK` | `GRAY_CLK_Pin` / `GRAY_CLK_GPIO_Port` | GPIO_Output |
| DAT | PC3_C | `GRAY_DAT` | `GRAY_DAT_Pin` / `GRAY_DAT_GPIO_Port` | GPIO_Input PullUp |
| 备用 | PC1 | `GRAY_UNUSED1` | `GRAY_UNUSED1_Pin` / `GRAY_UNUSED1_GPIO_Port` | GPIO_Input |
| 备用 | PC2_C | `GRAY_UNUSED2` | `GRAY_UNUSED2_Pin` / `GRAY_UNUSED2_GPIO_Port` | GPIO_Input |

平台层每次读取先用 `CLK=0` 保持约 `1 ms` 做帧同步，再按 bit0 到 bit7 从 `DAT` 连续读取 8 路数字量。当前读取极性为 `DAT != 0` 视为检测到黑线。

## 8. 后续硬件接入建议

1. 编码器：已使用 TIM1/TIM8 接入真实 AB 相计数；实车前应先跑 `MED_CAR_TEST_MODE_ENCODER` 确认左右轮方向和计数增量。
2. 灰度循迹：当前只改底层读取，保留原先颜色判断逻辑；后续如果要充分利用第 8 路，可升级 `xunxian()` 的权重/位置判断。
3. 药品检测：确定输入引脚后打开 `MED_CAR_ENABLE_DRUG_SENSOR`，并将 `MedicineCar_ReadDrugPresent()` 改为读取真实 GPIO。
4. 蓝牙和识别串口：建议新增独立 UART，不与 USART1 调试串口混用。
5. TFT 和按键：建议先完成小车运动闭环，再逐项启用显示和人工控制模块。
