## 1. 选用接口

- 串行通讯使用 `CLK` 和 `DAT` 两根信号线。
- `CLK` 由主控输出，`DAT` 由传感器输出、主控输入。
- 单个传感器只需要 2 路 GPIO；多个传感器可共用 `CLK`，每个传感器单独占用 1 路 `DAT`。
- 串行通讯输出内容与并行 1 到 8 路数字量一致，不包含 I2C 才有的模拟量、阈值配置、错误信息等扩展功能。

## 2. 数据位定义

- 一帧为 8 bit。
- 第 0 bit 对应第 1 路 `OUT1`。
- 第 1 bit 对应第 2 路 `OUT2`。
- 依此类推，第 7 bit 对应第 8 路 `OUT8`。
- 代码中可按 `gray_sensor_values[i] = (status >> i) & 0x01U` 映射到 `IO1..IO8`。

## 3. 读数时序

手册给出的读取原则：

- 每一位读取时，先将 `CLK` 拉低。
- `CLK` 为低电平时读取 `DAT`。
- 读取后将 `CLK` 拉高。
- `CLK` 高电平至少保持 `5 us`，保证传感器内部移位寄存器完成运算。
- 8 位读取必须在帧内连续完成，每个时钟间隔应明显小于 `1 ms`。

伪代码：

```c
uint8_t status = 0U;

for (uint8_t i = 0U; i < 8U; i++) {
    CLK = 0;
    if (DAT != 0U) {
        status |= (1U << i);
    }
    CLK = 1;
    delay_us(5);
}
CLK = 0;
```

## 4. 帧同步

- 传感器内部使用约 `1 ms` 的无时钟间隔作为“归零”同步条件。
- 每帧读完后应留出约 `1 ms`，让下一次读取从第 0 bit 重新开始。
- 如果不做帧间归零，理论上也能循环读取，但一旦错位，后续数据都可能错，因此项目代码应显式保留帧同步延时。
- 手册说明内部毫秒计时约 `0.8 ms` 即可产生同步脉冲，代码使用 `HAL_Delay(1U)` 即可满足要求。

## 5. GPIO 与电平兼容

- `CLK`：推挽输出即可，不受是否安装 `PULL` 跳线帽影响。
- `DAT`：主控输入。
- 若灰度传感器按官方推荐 `5 V` 供电，而 STM32 主控 IO 为 `3.3 V`，应安装传感器 `PULL` 开漏跳线帽并断电重启传感器。
- 开漏模式下 `DAT` 需要上拉到主控 IO 电压，STM32 侧建议配置为上拉输入；也可外接 `10 k` 到 `60 k` 上拉电阻到 `3.3 V`。
- 传感器与 STM32 必须共地。

## 6. 对当前工程的重构落点

- 当前上层循迹接口仍保持 `MedicineCar_ReadLineSensors(uint8_t values[8])` 和 `MedicineCar_ReadLineSensor(uint8_t index)`。
- 仅替换平台层 `gray_update_all()` 的底层采样方式：由旧的 `AD0/AD1/AD2 + OUT` 通道选择扫描，改为 `CLK/DAT` 串行读取 8 bit。
- 建议复用现有灰度 GPIO 资源：
  - `GRAY_CLK` / `PC0` 接串行 `CLK` 输出。
  - `GRAY_DAT` / `PC3_C` 接串行 `DAT` 输入。
  - `GRAY_UNUSED1` / `PC1`、`GRAY_UNUSED2` / `PC2_C` 暂不参与串行读取，可保留为空闲 GPIO 或后续重新分配。
- 现有 `xunxian()` 已通过 `MedicineCar_ReadLineSensors(gray)` 一次读取 8 路，适合直接承接新的串行读取缓存。
