# 2021 年电赛 F 题“智能送药小车”分析报告

## 0. 资料来源与总体判断

本报告基于目录中的 `赛题.png` 和 `2021F参考.pdf` 综合整理。赛题要求设计并制作智能送药小车，模拟医院药房与病房之间的药品送取作业。场地以红色实线作为走廊中心导航线，黑色实线表示墙体边界，黑白虚线表示药房或病房门口区域，病房号为 1~8 的黑色数字标识。

本队选择 STM32H743VIT6 作为主控板。建议总体方案采用：

> STM32H743VIT6 主控 + 巡线传感器阵列 + 独立视觉识别模块 + 电机驱动 + 药品检测 + 声光提示 + 双车无线通信

其中 STM32 负责实时控制、状态机、路径决策、运动控制、通信协调；视觉模块只负责数字识别并通过串口向 STM32 输出识别结果。这样可以降低 STM32 端图像处理压力，也便于在短时间内调通整车。

## 1. 赛题分析与三人合作最佳分工方式

### 1.1 任务核心

赛题核心不是单纯“巡线小车”，而是“识别目标病房号后，按照场地结构完成送药、等待卸药、自动返回”的闭环系统。基本任务要求单车完成近端、中部、远端病房送药并返回；发挥任务要求两台小车通过通信协同，避免碰撞、错车和超车。

主要流程如下：

1. 小车放置在药房门口区域，车头面向病房方向。
2. 小车识别目标病房号，数字范围为 1~8。
3. 装载约 200 g 药品后，小车检测到装药完成并自动启动。
4. 小车沿红色中心线巡线，根据目标病房和路口信息自动寻径。
5. 到达病房门口区域后停车，点亮红灯，等待人工卸药。
6. 卸药后红灯熄灭，小车自动返回药房。
7. 返回药房门口区域后停车，点亮绿灯。
8. 双车任务中，需要通过黄灯暂停、无线通信和路径互斥避免冲突。

### 1.2 难点排序

按工程风险排序，建议优先关注：

1. 巡线稳定性：红线识别、黑线避让、路口判断、门口虚线判断。
2. 路径状态机：不能只靠固定延时，必须根据目标病房号、路口计数、转向动作和门口检测推进状态。
3. 数字识别稳定性：药房处目标数字必须在 20 s 内识别，3~8 号随机设定时要避免写死路径。
4. 转向与停车精度：车头投影需要落在门口区域内，转弯过冲会直接导致任务失败。
5. 装卸检测闭环：装药自动启动，到房红灯停车，卸药后自动返回，回药房绿灯停车。
6. 双车协同：不是越快越好，首先要保证两车不会进入同一走廊相向错车、追尾或超车。

### 1.3 三人最佳分工

建议采用“软件主控 + 视觉/算法 + 硬件/调试”三人并行方式，而不是按模块平均分。原因是本题最后的成功取决于系统联调，必须有人持续维护主状态机和接口规范。

#### 成员 A：主控软件与系统集成负责人

职责：

- 建立 STM32H743VIT6 工程。
- 编写电机 PWM、编码器测速、巡线采样、按键、LED、蜂鸣器、装卸检测等底层驱动。
- 设计整车状态机和路径决策逻辑。
- 统一各模块接口，维护全局任务流程。
- 负责单车近端、中部、远端完整闭环调通。

交付物：

- STM32 主工程。
- `motor`、`line_sensor`、`motion`、`car_fsm`、`path_planner` 等核心代码。
- 状态机流程图、关键调试参数表。

#### 成员 B：数字识别与通信负责人

职责：

- 负责 OpenMV、K210、K230 或其他视觉模块选型和调试。
- 实现 1~8 号数字识别，多帧投票输出稳定结果。
- 设计视觉模块到 STM32 的串口协议。
- 负责双车无线通信协议，包括消息帧、重复发送、ACK 或超时保护。
- 双车协同时，与成员 A 对接状态机事件。

交付物：

- 视觉模块程序。
- UART 协议文档。
- 数字识别成功率测试记录。
- 双车通信消息表与静态通信测试结果。

#### 成员 C：机械硬件、电源与现场调试负责人

职责：

- 搭建小车底盘、电机驱动、电源、传感器支架、摄像头支架、药品托盘。
- 保证车体尺寸不大于 25 cm × 20 cm × 25 cm。
- 负责电机供电、主控供电、共地、抗干扰和布线可靠性。
- 标定巡线传感器阈值，记录不同光照下红/黑/白读数。
- 负责场地搭建、路线测试、计时和问题复盘。

交付物：

- 硬件连接表。
- 传感器安装位置图。
- 电池电压、速度、转向参数、失败原因记录表。

### 1.4 协作节奏建议

第 1 天先完成底盘运动、传感器读取、数字识别单项验证；第 2 天完成单车近端和中部闭环；第 3 天攻克远端随机识别和返回；第 4 天再做双车协同与速度优化。不要等所有硬件装完再写状态机，应从第一天就建立可运行的工程骨架和模拟输入接口。

## 2. 项目搭建思路

### 2.1 总体技术路线

推荐使用“两轮差速底盘 + 红线巡迹 + 路口事件 + 目标房号状态机”的路线。

小车运动不建议依赖固定时间开环完成，因为电池电压、地面摩擦、车轮差异和载重都会使固定时间转向误差累积。较稳妥的方法是：

- 直线段：根据巡线传感器计算偏差，使用 PD 或 PID 调整左右轮速度。
- 路口处：用连续多帧检测确认十字/T 型/门口虚线事件。
- 转向处：低速进入，执行“定角转向 + 重新捕获红线”。
- 停车处：检测门口虚线或目标门口标志后停车，再用短距离微调。
- 返回时：使用路径记忆，送药时把关键转向压入栈，返回时逆序执行。

### 2.2 路径规划思路

病房可分为三类：

- 近端病房：1、2 号，位置固定，距离药房最近。
- 中部病房：3、4 号，位置在中部支路，测试时数字可能随机设置。
- 远端病房：5、6、7、8 号，路径更长，支路和门口识别难度更高。

推荐状态机不要直接写成“目标 1 走几秒左转，目标 2 走几秒右转”，而应抽象为：

- 目标房号 `target_room`
- 当前路段 `segment_id`
- 已经过路口计数 `cross_count`
- 当前方向 `heading`
- 目标门口左右信息 `door_side`
- 是否已经到达目标区域 `target_zone_reached`
- 送药路径栈 `path_stack`

当小车经过路口时，状态机根据 `target_room` 和 `cross_count` 决定继续直行、左转或右转。进入支路后，通过门口虚线和数字/位置关系确认停车点。

### 2.3 推荐实现阶段

1. 裸车运动阶段：电机正反转、PWM 调速、直行 50 cm、原地转向。
2. 巡线阶段：只沿红线直行，能稳定跑 1 m 以上。
3. 事件检测阶段：识别十字路口、T 型路口、门口虚线和黑线边界。
4. 输入识别阶段：药房处识别目标数字，串口发送给 STM32。
5. 单车闭环阶段：完成装药启动、到房红灯、卸药返回、药房绿灯。
6. 远端泛化阶段：处理 5~8 号随机设定，不写死单一远端位置。
7. 双车协同阶段：加入无线通信、暂停点、继续指令和超时保护。
8. 速度优化阶段：分段速度规划，直线快、路口慢、转向慢、门口慢。

## 3. 项目所需软件与软件代码架构

### 3.1 开发软件建议

主控 STM32H743VIT6：

- STM32CubeMX：配置时钟、GPIO、TIM、ADC、UART、DMA、中断。
- Keil MDK 或 STM32CubeIDE：编译、下载、调试。
- ST-Link 工具：下载程序、串口日志辅助调试。
- 串口调试助手：查看视觉模块、无线模块、主控日志。
- Git：保存不同阶段可用版本，避免联调时把稳定版本改坏。

视觉模块：

- OpenMV IDE，或 K210/K230 对应 IDE 与 SDK。
- 若使用摄像头模块进行模型识别，需要准备数字样本、阈值参数或轻量模型。

测试记录：

- Excel/WPS 表格：记录阈值、速度、转向时间、识别成功率、单项耗时。
- 手机录像：复盘转向过冲、压线、误识别和碰撞问题。

### 3.2 STM32H743VIT6 外设规划

建议外设分配如下：

| 功能 | 外设建议 | 说明 |
|---|---|---|
| 左右电机 PWM | TIM1/TIM8/TIM3 PWM | H743 定时器资源充足，优先使用高级定时器或通用定时器 |
| 编码器测速 | TIM2/TIM4 编码器模式 | 有编码器电机时用于速度闭环和定距转向 |
| 巡线传感器 | GPIO/ADC/I2C | 数字灰度阵列用 GPIO，模拟灰度用 ADC，颜色传感器可能用 I2C |
| 视觉模块通信 | UART | 推荐 115200 或更高波特率，使用帧头和校验 |
| 无线通信 | UART/SPI | 蓝牙/无线串口用 UART，NRF24L01 用 SPI |
| 药品检测 | GPIO/ADC | 微动开关、红外对射、压力传感器均可 |
| 声光提示 | GPIO/PWM | 红绿黄 LED，蜂鸣器可用 GPIO 或 PWM |
| 调试日志 | UART/USB CDC | 输出状态机状态、目标房号、传感器事件 |

H743 性能较强，但要注意 D-Cache、DMA 缓冲区一致性和 3.3 V 电平兼容。若不熟悉 Cache/DMA，早期可以先不用复杂 DMA，把系统调通后再优化。

### 3.4 核心模块职责

`motor`：

- 封装左右轮方向和 PWM。
- 提供 `Motor_SetLeft()`、`Motor_SetRight()`、`Motor_StopAll()`。

`encoder`：

- 读取左右轮编码器计数。
- 计算速度、里程和转向角度估计。

`line_sensor`：

- 采集红/黑/白状态。
- 计算巡线偏差 `line_error`。
- 输出事件：`LINE_FOUND`、`CROSS_FOUND`、`T_FOUND`、`DOOR_FOUND`、`BLACK_LIMIT`。

`motion_ctrl`：

- 实现直行巡线、低速转弯、停车、重新找线。
- 内部使用 PD/PID 调整左右轮速度。

`vision_protocol`：

- 接收视觉模块数据帧。
- 输出稳定目标房号、识别置信度、数字位置。
- 建议多帧投票，例如连续 5 帧中 3 帧一致才确认。

`load_detect`：

- 检测药品是否存在。
- 做防抖处理，避免装药瞬间抖动误启动。

`path_planner`：

- 根据目标病房号、路口计数、当前位置决定动作。
- 记录送药路径，返回时逆序执行。

`car_fsm`：

- 整车主状态机，管理识别、等待装药、送药、到房、等待卸药、返回、完成。

`car_comm`：

- 双车通信，处理暂停、继续、返回、到达等消息。
- 需要重复发送、超时重发和简单校验。

### 3.5 主状态机建议

建议主状态如下：

```c
typedef enum {
    CAR_IDLE = 0,
    CAR_RECOGNIZE_TARGET,
    CAR_WAIT_LOAD,
    CAR_GO_MAIN_LINE,
    CAR_CHECK_INTERSECTION,
    CAR_TURN_TO_ROOM,
    CAR_FIND_DOOR,
    CAR_WAIT_UNLOAD,
    CAR_RETURN_MAIN_LINE,
    CAR_RETURN_PHARMACY,
    CAR_FINISH,
    CAR_ERROR
} CarState_t;
```

状态推进原则：

- 所有灯亮状态必须停车。
- 所有事件必须连续确认，不能单帧触发。
- 所有等待状态都要有超时和错误提示。
- 关键状态变化必须串口打印，便于现场排错。
- 不建议在状态机中大量使用 `HAL_Delay()`，应使用系统 tick 或定时任务。

### 3.6 控制周期建议

若不使用 RTOS，可用主循环 + 定时器节拍：

| 周期 | 任务 |
|---|---|
| 1 ms | 系统 tick、按键/开关防抖计数 |
| 5 ms | 电机速度环、编码器采样 |
| 10 ms | 巡线传感器采样、偏差计算 |
| 20 ms | 状态机调度、事件确认 |
| 50 ms | 串口日志、通信心跳 |
| 100 ms | LED/蜂鸣器状态刷新 |

H743 可以运行 FreeRTOS，但本题时间紧时不强依赖 RTOS。裸机状态机更容易排查实时问题；若队伍熟悉 RTOS，可将运动控制、传感器、通信和状态机拆成任务。

## 4. 硬件模块搭建简要思路

本题硬件以“可靠、可调、易维修”为优先级，软件为主，硬件服务于稳定识别和稳定运动。

### 4.1 底盘与电机

- 使用两轮差速底盘，前/后加万向轮或牛眼轮支撑。
- 尽量选择带编码器的减速电机，方便速度闭环和转向一致性控制。
- 车体尺寸控制在 25 cm × 20 cm × 25 cm 内，传感器和摄像头支架也要计入尺寸。
- 重心靠近两驱动轮中间，药品托盘居中，避免装 200 g 药品后偏航。

### 4.2 电机驱动与供电

- 电机驱动可选 TB6612、DRV8833、BTS7960 等，按电机电流选择。
- 电机电源与 STM32 电源分开稳压，但必须共地。
- STM32 使用稳定 5 V/3.3 V 供电，电机端增加足够滤波电容。
- 电机线和传感器线尽量分开走线，减少电机噪声影响灰度/颜色采样。

### 4.3 巡线传感器

推荐使用前置 8 路或更多路灰度/颜色传感器阵列：

- 安装在车头前方，离地高度固定。
- 传感器横向覆盖宽度要略大于红线宽度。
- 中间传感器用于红线跟踪，两侧传感器辅助判断偏离和黑线边界。
- 用连续计数判断门口虚线和路口，避免单次采样误判。

如果预算允许，线性 CCD 或摄像头巡线效果更强，但调试复杂度更高。短周期备赛建议优先选择灰度/颜色阵列。

### 4.4 数字识别模块

建议视觉模块独立完成数字识别：

- OpenMV：易用、调试快，适合传统图像处理或模板匹配。
- K210/K230：AI 推理能力强，适合训练轻量数字识别模型。
- 摄像头安装角度要固定，药房识别时数字纸张距离和角度尽量统一。
- 输出给 STM32 的数据应简单可靠，例如：

```text
HEAD  SRC  CMD  ROOM  CX  CONF  CHECK
0xAA 0x01 0x10 0x05 0x64 0x58 0x??
```

STM32 端只接收“目标房号、有效标志、置信度、可选位置”，不要在主控里做复杂图像处理。

### 4.5 药品检测

可选方案：

- 微动开关：最简单，装药压下开关。
- 红外对射/反射：无接触，但受环境光和药品形状影响。
- 压力传感器：能区分载重，但需要 ADC 标定和滤波。

建议首选微动开关或限位开关，可靠性高、调试快。软件上必须做防抖，例如连续 200 ms 检测到有药才认为装药完成，连续 200 ms 无药才认为卸药完成。

### 4.6 指示灯与蜂鸣器

- 红灯：到达病房，停车等待卸药。
- 绿灯：返回药房，任务完成。
- 黄灯：双车协同中小车 2 暂停等待。
- 蜂鸣器：可用于识别成功、错误状态、通信超时提示。

必须满足：任一指示灯处于点亮状态时，小车必须是停止状态。

### 4.7 双车通信

可选无线串口、蓝牙、NRF24L01。短期实现建议无线串口，因为对 STM32 来说就是 UART，协议简单。

建议消息：

| 消息 | 含义 |
|---|---|
| `C1_AT_ROOM` | 小车 1 已到达病房 |
| `C1_UNLOADED` | 小车 1 已卸药，准备返回 |
| `C1_RETURNING` | 小车 1 正在返回药房 |
| `C2_PAUSE_OK` | 小车 2 已到暂停点并停车 |
| `C2_GO` | 小车 2 可以继续前进 |
| `C2_AT_ROOM` | 小车 2 已到达病房 |

通信协议必须支持重复发送和超时处理，避免单包丢失导致小车永久等待。

## 5. 赛题调试注意事项

### 5.1 调试顺序

强烈建议按以下顺序调试：

1. 电机方向和 PWM：确认左/右轮正反转正确。
2. 低速直行：验证机械偏差、电源压降和载重影响。
3. 巡线直行：只跑红线，不处理路口。
4. 路口/虚线事件：低速或手推采样，确认事件触发位置。
5. 转向找线：每种左转/右转都要能重新捕获红线。
6. 数字识别：在药房处识别 1~8，做多帧投票。
7. 装卸检测：验证装药启动、卸药返回。
8. 单车近端：先完成 1/2 号。
9. 单车中部：再完成 3/4 号。
10. 单车远端：最后完成 5~8 号。
11. 双车静态通信：两车不动时先把所有消息测通。
12. 双车低速协同：低速完整跑通后再提速。

### 5.2 常见问题与处理

| 问题 | 可能原因 | 处理建议 |
|---|---|---|
| 小车巡线左右摇摆 | P 参数过大、速度过快、传感器太靠前或太靠后 | 降速，减小 P，加入 D，调整传感器位置 |
| 经过路口误判 | 单帧触发、阈值不稳、红线宽度变化 | 连续多帧确认，加入最小间隔时间 |
| 转弯后找不到线 | 转角过大/过小、速度太快、传感器覆盖不足 | 转弯末端低速扫线，捕获红线后再进入巡线 |
| 到门口停不准 | 门口虚线识别不稳、停车惯性大 | 门口区域提前降速，检测到门口后短刹车 |
| 装药后不启动 | 开关抖动、药品没压到检测点 | 修改托盘结构，增加防抖和状态打印 |
| 卸药后误判 | 传感器滞后或药品残留压住开关 | 调整开关位置，连续确认空载 |
| 视觉识别错误 | 光照变化、数字倾斜、距离不固定 | 固定识别距离，多帧投票，加入置信度阈值 |
| 双车通信卡死 | 丢包、状态未同步、未设超时 | 重复发送，ACK 应答，超时进入安全停车 |
| H743 串口 DMA 数据异常 | Cache 一致性问题 | 先不用 DMA，或把 DMA 缓冲区放到非缓存区并做 Cache 维护 |

### 5.3 参数记录建议

每次调试要记录参数，避免“感觉调车”。建议记录：

- 电池电压。
- 左右轮基础 PWM。
- 巡线 P/D 参数。
- 红/白/黑传感器阈值。
- 路口确认帧数。
- 门口确认帧数。
- 转向 PWM、转向时间或编码器计数。
- 近端、中部、远端送达时间和返回时间。
- 失败原因和修改结果。

### 5.4 现场验收策略

- 优先保证基本三项稳定拿分，再追求发挥部分。
- 直线段可以提速，路口、转向、门口必须降速。
- 每次正式测试前只允许一次复位，装药后立即计时，不能依赖人工干预。
- 小车投影不能持续落在黑实线上，连续压黑线超过限制或整车越线风险极高。
- 双车协同时宁可等待，也不能在同一走廊错车或超车。
- 所有灯亮时必须停车，这一点要直接写进状态机约束。

## 6. 推荐最终方案摘要

最终建议方案如下：

- 主控：STM32H743VIT6。
- 底盘：两轮差速 + 编码器电机 + 万向轮。
- 巡线：8 路灰度/颜色传感器阵列，识别红线、黑线、门口虚线。
- 视觉：OpenMV/K210/K230 独立识别 1~8 数字，UART 输出目标房号。
- 药品检测：微动开关或压力传感器，软件防抖。
- 电机控制：PWM + 方向控制，编码器速度闭环。
- 路径控制：目标房号 + 路口计数 + 路径栈返回。
- 软件架构：底层驱动、传感器层、运动控制层、路径规划层、整车状态机、通信层分层实现。
- 双车协同：无线串口通信，消息重复发送，暂停点停车黄灯，收到继续指令后运行。

本题最稳的攻关路线是：先用低速稳定完成单车闭环，再通过参数和分段速度规划优化时间；双车部分必须在单车稳定之后再加入，否则故障来源会混在一起，调试效率很低。
